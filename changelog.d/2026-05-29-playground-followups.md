## fix: playground-tests follow-ups — actions-pinned regex + dev-stage wording

| Type | Summary |
|---|---|
| fix | **`scripts/check-actions-pinned.sh` — inspect the standard step form.** The match regex was `^\s+uses:\s+[^.][^/]+/`, whose leading-whitespace anchor only matched the dash-less reusable-workflow caller form (`jobs.<j>.uses:`). The far more common step form `      - uses: owner/repo@ref` (list-item dash) never matched, so **step-level action uses were silently never checked** for SHA-pinning — a supply-chain gate that wasn't actually gating most `uses:` lines. Surfaced by the stage-2 BATS tests of the playground-tests dev. Fix: allow an optional list-item dash — `^\s+(-\s+)?uses:\s+[^.][^/]+/`. The `sed` ref-extraction already handled the dash form, and `[^.]` still skips local `./...` actions in both forms. Verified the fixed script PASSes on every current factory workflow (all already pinned — no CI break) and on the system-template workflows. |
| test | **`scripts/tests/check-actions-pinned.bats` — lock the fix in.** Replace the header's "latent bug" note with the corrected behavior, and add step-form cases: pinned `- uses:` → PASS, local `- uses: ./…` → ignored, `@v5` tag → FAIL, short SHA → FAIL. These fail on the old regex and pass on the fixed one (regression lock). Reusable-caller cases retained. Suite: **`bats scripts/tests/*.bats` = 32/32 PASS**. |
| fix | **`dev-stage.md` (a.1) wording — resolve a self-contradiction.** Step (a.1) said "do NOT proceed to bookkeeping until the Playground check is green," but step (b) writes bookkeeping **in the same commit as the code** — so verification can't precede bookkeeping. Reworded (a.1): code + bookkeeping (b) ride in one commit, push, wait for CI, and don't **report the stage done (c) / move to the next (d)** until Playground is green. Safety Rule #7 changed from "never proceed to bookkeeping with a red Playground" to "never close a stage or move to the next with a red Playground." Mirrored byte-identically into `templates/system/.claude/commands/dev-stage.md`. |
