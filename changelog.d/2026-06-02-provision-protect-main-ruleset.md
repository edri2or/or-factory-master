### Changed
- `.github/workflows/provision-system.yml`: the "Branch protection on main" step now applies the **same `protect-main` ruleset the factory enforces on itself** (via `scripts/ensure-protect-main-ruleset.sh` with the 4 system contexts) instead of the legacy classic branch-protection API (`PUT /branches/main/protection`). Every newly-provisioned system is now born with a server-side lock that blocks direct pushes to `main` for **everyone including the broker App** (it is not a repo admin, so it does not match the ruleset's admin bypass actor), plus no force-push and no deletion. The step runs after all scaffold pushes, and the unscoped broker token minted earlier already carries `administration:write`, so no token change was needed. Existing systems are unaffected (they keep their classic protection until re-hardened); `or-adhd-agent` already carries the ruleset.
