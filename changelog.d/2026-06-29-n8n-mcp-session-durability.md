- **Group 1 (idle-reaping mitigation) ‚Äî n8n-mcp-session-durability.** Pinned the n8n-mcp sidecar's idle-session reap window to `SESSION_TIMEOUT_MINUTES="120"` in `scripts/render-mcp-service-yaml.sh` (n8nmcp env block, after `LOG_LEVEL`). The sidecar previously ran the upstream default of 5 min (was 30 before czlonkowski/n8n-mcp v2.33.5), so an operator dev session on `/n8n/<system>/mcp` that paused past 5 min got the intermittent "Session not found / expired" (HTTP 400, JSON-RPC -32001) ‚Äî Layer A of the two root causes. 120 min is generous vs the documented 15‚Äì30 floor; the proxy's transparent re-init still recovers even past it. `N8N_MCP_IMAGE` verified unchanged at `2.51.2` (already at the GHSA-jxx9-px88-pj69 security floor ‚Äî a version bump is a separate, separately-proven change). Proven by a local render asserting `SESSION_TIMEOUT_MINUTES=120` inside the `n8nmcp` container; shellcheck-clean. The durable cross-instance store (Layer B) follows in Group 2.

- **Group 2 (durable cross-instance session store, Layer B) ‚Äî n8n-mcp-session-durability.** The n8n-mcp proxy remembered each client's `initialize` only in instance RAM (`n8n-mcp-proxy.ts`), so any Cloud Run instance replacement (DEPLOY_NONCE revision roll, maintenance, OOM) wiped it and transparent recovery failed ‚Äî the client saw the raw "Session not found". Added a durable mirror so recovery survives replacement:
  - **New `services/mcp-server/src/session-store.ts`** ‚Äî a Firestore-backed `SessionBackend` accessed via the REST API + the ambient runtime-SA ADC token (`getGcpAccessToken` from `gcp-client.ts`), mirroring the house "bare fetch, no `@google-cloud/*`" style ‚Äî **zero new npm deps**. Stores only `{initBody, upstreamSid, system}` (NO secrets ‚Äî the per-tenant `x-n8n-url`/`x-n8n-key` are injected per-request, never serialized). Every method is **fail-open** with a 4 s timeout, so a store outage degrades to today's RAM-only behavior, never worse. 1 h TTL (server-side Firestore TTL + an app-side `expireAt` check). Pure helpers (`buildSessionDocPath`, `encode/decodeSessionDoc`, the tenant-guarded `resolveStoredSession`) are exported for network-free unit tests.
  - **`n8n-mcp-proxy.ts` wiring (4 edits)** ‚Äî `rememberSession` write-through to the store; `reinitUpstream` reads the store on a RAM-map miss and rehydrates **only a same-tenant record** (cross-tenant replay is structurally impossible); the recovery gate no longer requires `sessions.has(clientSid)` so recovery fires on an empty post-replacement map; client-DELETE clears both tiers. `recovering` dedupe + `SESSION_CAP` eviction untouched. Injectable `setSessionBackend` test seam.
  - **`scripts/render-mcp-service-yaml.sh`** ‚Äî gateway env `SESSION_STORE_PROJECT="or-factory-master-control"` + kill-switch `SESSION_STORE_ENABLED="1"` (`"0"` = instant RAM-only rollback, no code change). No new secret ‚Äî auth is ambient ADC.
  - **`.github/workflows/deploy-mcp-server.yml`** ‚Äî idempotent, best-effort "ensure Firestore session store" step: enable `firestore.googleapis.com`, create the default Native database in `me-west1` (a supported Firestore Native location; the location is immutable) if missing, set the TTL policy on `n8n-mcp-sessions.expireAt`, and grant the runtime SA `roles/datastore.user`. Best-effort by design ‚Äî a provisioning gap can't fail the deploy because the store is fail-open.
  - **Proof.** 12 new pure unit tests (`services/mcp-server/test/session-store.test.mjs`): encode/decode round-trip, **secret-safety** (the serialized record contains no `x-n8n-key`/`apiKey`/`Bearer`), app-side TTL expiry, the tenant guard, and the fail-open contract against a throwing fake backend ‚Äî `node --test` 150/150 green. The live **acceptance gate** is the extended `scripts/n8n-mcp-smoke.py` (phases `full`/`init`/`recover`) + `n8n-mcp-smoke.yml`: it establishes a session, **forces a real gateway instance replacement** (a new revision wipes the min=max=1 instance's RAM ‚Äî the exact production event), then reuses the same session id and asserts the call still succeeds ‚Äî which can only happen if the durable store rehydrated it. To run after the Group 2 deploy lands.
