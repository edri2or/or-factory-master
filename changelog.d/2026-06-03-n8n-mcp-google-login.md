## feat: operator "Login with Google" for the MCP gateway (gateway-native Google IdP)

| Type | Summary |
|---|---|
| feat | Replace the gateway's admin-secret OAuth login with **Login with Google** so the operator never handles a secret, without the heavyweight Cloudflare-Access-+-Tunnel route (an always-on `cloudflared` that would break Cloud Run scale-to-zero — rejected). The gateway stays the OAuth Authorization Server and delegates *identity* to Google: `/oauth/authorize` redirects to Google (the MCP client's PKCE/state stashed in a `pendingAuth` map keyed by a random server state), and a new `/oauth/callback` exchanges the Google code, verifies the email against `OAUTH_ALLOWED_EMAILS`, then mints the gateway's existing auth code bound to the client's PKCE. New `src/google-oauth.ts` (config check, authorize-URL builder, code exchange + email extraction). Falls back to the admin-secret form when Google isn't configured, so the deploy is safe before the Google client exists; the `/token` (admin) and `/oidc/token` (CI) machine paths are unchanged. New secrets `google-oauth-client-{id,secret}` (placeholder-seeded in `deploy-mcp-server.yml`; real values added once the operator creates the Google OAuth client — the one manual step) and a new `OAUTH_ALLOWED_EMAILS` env wired through the service renderer. Unit-tested (`test/google-oauth.test.mjs`); `tsc` + full suite green. |
