## feat: wake the watching session on CI-green via a PR-review relay

| Type | Summary |
|---|---|
| feat | New `notify-routine-ci-green.yml` (`workflow_run` on the five CI workflows [`Changelog Check` / `Pipeline Tests` / `Secret Scan` / `Supply Chain Check` / `Playground Tests`] + manual). A Claude Code session subscribed to a PR is pushed events for CI *failures* and review comments but **not** for CI *success*, so a green PR never wakes the watching session on its own. A submitted PR *review* **is** pushed to that session — so when CI completes, this posts a short review (`event=COMMENT`, via `gh api` with the default `GITHUB_TOKEN`) that wakes the session immediately, no timer/poll. Proven live in the sibling system `or-agents` (2026-07-24). Because the factory splits CI across five workflows (unlike or-agents' single umbrella), the step first re-reads **every check-run on the PR's head commit** and posts only when they are all green — so the wake is honest and fires exactly once (only the last-finishing run sees all-green). `continue-on-error` throughout; it never reddens CI. Non-blocking: **not** added to any `protect-main` required-contexts set. |
| feat | Same workflow added to the gold template `templates/new-system/.github/workflows/notify-routine-ci-green.yml` (fully generic — no `__SYSTEM_NAME__` placeholder to fill; listens to a clean sibling's `Pipeline Tests` + `Secret Scan`), so every future `/new-system` sibling inherits the CI-green wake bridge out of the box. `or-agents` already carries its own copy and is untouched. |
| docs | `CLAUDE.md` updated — a row in the **Workflows (live)** table for `notify-routine-ci-green.yml`, and a note in the **Sibling-system builder** section that the template now ships it to every future sibling. |
