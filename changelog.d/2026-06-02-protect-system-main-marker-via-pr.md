### Fixed
- `.github/workflows/protect-system-main.yml`: write `.bootstrap-complete` through a PR instead of a direct push to `main`. The broker App is not a repo admin on system repos, so it does not match the protect-main ruleset's `bypass_actors` — a direct `PUT .../contents` to `main` is rejected with 409 ("Changes must be made through a pull request") once the ruleset is active. The marker step now branches → commits → opens a PR → retries the squash-merge until the system's own CI is green (the merge endpoint enforces the ruleset itself, so a 200 is the CI gate — no `checks:read` needed), the same pattern `prove-on-test-system.yml` and `refresh-system-agents.yml` already use. The minted token gains `pull_requests:write` (alongside `contents:write`) when `write_bootstrap_marker=true`. This is consistent with hardening's intent that even the broker goes through a PR — no ruleset bypass is added.
