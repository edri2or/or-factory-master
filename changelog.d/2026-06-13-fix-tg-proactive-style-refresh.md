## fix: the two daily bot automations that never once succeeded — tg-proactive SQL + style-refresh empty input

| Type | Summary |
|---|---|
| fix | A live state-audit of or-edri-4's n8n (25 workflows) found **two active workflows that have failed on every run and never once succeeded**, both rooted in `templates/system/workflows/n8n/`. **(1) `tg-proactive` (daily 08:00 summary)** failed at the `Aggregate Stats` Postgres node with `column "5786217215" does not exist`. Root cause: that one query built the chat session id as `'tg:' || {{ JSON.stringify('@@CHAT_ID@@') }}` — after `configure-agent-router.yml` substitutes the raw chat id, `JSON.stringify` wraps it in **double quotes** (`"5786217215"`), which Postgres reads as an **identifier (column name)**, not a value. This `JSON.stringify('@@CHAT_ID@@')` was the **only** occurrence across all workflows; five other files (postgres-named-queries, github-readonly, railway-readonly, request-write-action, style-refresh) build the same `session_id` correctly as `'tg:' || '@@CHAT_ID@@'`. Fix: align tg-proactive's `Aggregate Stats` to that proven pattern (drop `JSON.stringify`, single-quote the placeholder). **(2) `style-refresh` (daily 03:00 style learner)** failed at the `Extract Style` chainLlm node with OpenRouter `400 Bad request`. The model `anthropic/claude-haiku-4.5` was **verified available** on OpenRouter (so the model — also used by code-agent, tg-proactive's Summarize Day, and agent-router's Synthesis — is not the fault). Root cause: `Extract Style` sends the transcript of the last 50 messages as the user prompt; on a fresh/low-traffic system the transcript is empty, and an empty user message is a 400 — which is exactly why it never succeeded. Fix: in the `Collect Messages` Code node, `return []` when no message content was collected, so the LLM node is skipped on empty input (the run ends green without updating the style profile — correct, there is nothing to learn from zero messages). Template-only; golden refreshed via `scripts/check-system-golden.sh --update` (two manifest lines). Reaches systems provisioned after merge; or-edri-4 gets it live via `refresh-system-agents.yml` + `configure-agent-router.yml`, with the definition of done being a real live `success` execution of both workflows, not a green job. |
