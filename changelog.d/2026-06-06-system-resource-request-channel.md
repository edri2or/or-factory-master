## feat: system resource-request channel — Stage 1 (deterministic request gate)

| Type | Summary |
|---|---|
| feat | First stage of the system → broker resource-request channel (a human-gated twin of the OIL loop that lets a provisioned system *ask* the broker to create a secret or grant an allowlisted IAM role it cannot do itself). Adds `scripts/validate-system-request.sh` — a pure, network-free deterministic gate that decides whether a request is legitimate before the broker ever acts: validates `request_type` (`secret`/`iam`), refuses control projects + `factory-test-25`, enforces the factory id shape on `system_name`/`gcp_project`, and locks grant targets to the system's own `deploy-sa`+`runtime-sa`. `secret` requests must be a safe id and not a super-credential (mirrors the EXCLUDE set in `copy-generic-secrets.sh`, plus privileged-keyword refusal); `iam` requests are default-deny against a curated, non-escalating role allowlist (owner/editor/`iam.*`/`*.admin`/`serviceusage.*` hard-refused). Proven by `scripts/tests/validate-system-request.bats` (30 cases, auto-discovered by `playground-tests.yml`); shellcheck clean. Behind a human ✅ — no fulfillment yet (Stage 2+). Tracked in `devplans/system-resource-request-channel.md`. |
