## fix: register→configure ordering preflight — App registered but secrets missing is now loud (master-system-integrity Stage 2, B3)

| Type | Summary |
|---|---|
| fix | The system `configure-agent-router.yml` reads `github-app-{id,installation-id,private-key}` from Secret Manager and installs the `github_readonly` tool only when all three are present. Until now two very different states collapsed to the **same silent warning**: (a) the App is simply **not registered yet** (legitimate — `register-system-app.yml` is a separate manual step), and (b) the App **was registered but its `github-app-*` secrets never reached SM** — a real ordering break that silently strips a live capability with the job staying green. **Fix (single file, `templates/system/.github/workflows/configure-agent-router.yml`; additive):** expose `APP_ID_VAR=${{ vars.APP_ID }}` (the repo variable `register-system-app.yml` writes once the App is registered) to the configure step, and add a preflight in the github-app block — `APP_ID` set **and** any `github-app-*` secret missing → `GH_APP_BROKEN`. That feeds a **critical** row into the existing B4 damage report (STEP_SUMMARY 🩺 + one `emit-event.sh` Telegram alert + `exit 1`), while `APP_ID` empty stays a green **warning** (a clean pre-registration birth is never falsely RED'd). No early exit — configure still wires router/telegram/mcp (all independent of `github_readonly`), then the damage report exits 1 at the end; the existing B4 warning row is guarded (`&& [ -z "$GH_APP_BROKEN" ]`) so a broken state shows exactly one critical row, not a duplicate. **Proof:** `scripts/tests/github-app-preflight-test.sh` (new, in the Playground tests job, 5/5) — registered+secrets-missing → RED, registered+partial-secrets → RED, not-registered → green warning, healthy → no row, broken suppresses the warning row. Golden refreshed (`scripts/check-system-golden.sh --update`). Template-only — reaches systems provisioned after merge; or-edri-4 gets it live via `refresh-system-agents.yml` + configure (the healthy path stays green; the broken `configure-before-register` path is proven deliberately at Stage 4 Day-0). |
