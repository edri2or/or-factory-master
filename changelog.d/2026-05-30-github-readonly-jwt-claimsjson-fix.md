## fix: github-readonly — JWT node payload field is claimsJson, not jsonPayload

| Type | Summary |
|---|---|
| fix | Second live-test follow-up to ops-agent-live-telemetry. After the `$getWorkflowStaticData` fix let `github_readonly` run, a fresh test system (`factory-test-023`) showed the ops-agent correctly calling the tool but GitHub returning **401** on every REST call. Root cause (confirmed against the n8n JWT node source, `packages/nodes-base/nodes/Jwt/Jwt.node.ts`): the `Sign JWT` node's JSON payload parameter is named **`claimsJson`**, but the workflow used `jsonPayload` — an unknown key the node ignored, so it signed an **empty** App JWT (no `iss`/`iat`/`exp`). GitHub rejected it, the installation-token mint returned 401, and the REST calls failed with 401. Renamed `jsonPayload` → `claimsJson` in `templates/system/workflows/n8n/github-readonly.json` (one key; `useJson:true` and the `jwtAuth` RS256 credential were already correct — algorithm comes from the credential per the node source). Routing, the `$getWorkflowStaticData` cache fix, and the ops-agent's tool wiring were all verified working live on the same system; `railway_readonly` returns real data. Template-only. |
