## Stage 111 ŌĆö fix: postgres-named-queries needs a defensive Normalize Code node ŌĆö Stage 110's workflowInputs fix alone wasn't enough

| PR | Type | Summary |
|---|---|---|
| TBD | fix | Stage 110 changed `templates/system/workflows/n8n/postgres-named-queries.json`'s `executeWorkflowTrigger` from `inputSource:"passthrough"` to `inputSource:"workflowInputs"` with `values:[{name:"query_name",type:"string"}]`, hoping the LLM's `query_name` would land at `$json.query_name`. Live verification on `factory-test-tgbot13` shows that didn't happen: 3 postgres-named-queries executions (22, 26, 27) ŌĆö each ~10-30 ms ŌĆö all routed straight to the Switch's `Unknown Query` fallback, never touching any Postgres branch. The LLM is calling the tool, but the args still aren't reaching `$json` in the shape the Switch expects (either `workflowInputs` isn't surfacing them on this n8n version + toolWorkflow v2.1 combination, or the args arrive at a different path like `$json.input.*`, `$json.query.*`, or `$json.arguments`). Fix without further blind guessing: new **Normalize Input** Code node between trigger and Switch ŌĆö extracts `query_name` defensively from any of the observed shapes (`raw.query_name`, `raw.input.query_name`, `raw.query.query_name`, `JSON.parse(raw.query).query_name`, `JSON.parse(raw.arguments).query_name`), and **always emits `_raw_input: <the original $json>`** so the next test run will reveal the actual shape via the `Unknown Query` Set node's new `_debug_received` field if the normalization still misses. Switch now reads `={{ $json.query_name }}` from Normalize Input's output (a no-op rename ŌĆö `$json` is just the previous node's data); Format Output reads `$('Normalize Input').first().json.query_name`; Unknown Query reads the same and adds `_debug_received` to its response payload. Idempotent + soft-fail: when the LLM actually provides `query_name` correctly at any known path, the Switch matches and the right Postgres query runs; when it doesn't, the LLM gets an error response with the raw payload it sent ŌĆö much easier to diagnose than the current "Unknown Query" black box. Per CLAUDE.md `templates/system/` edits reach only newly-provisioned systems; verification = same 3 Telegram messages on a fresh test (or a configure re-run on a system whose template would update, though tgbot13's already-installed subworkflow won't auto-update). |
